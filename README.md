# umanggarg.dev

Personal portfolio + production-grade chat agent. Live at **[umanggarg.dev](https://umanggarg.dev)**.

Built with Next.js 16 (App Router), Tailwind CSS 4, the Vercel AI SDK, and Anthropic Claude. The portfolio side is editorial-styled. The chat side is a production-grade tool-using agent: embedding RAG, Langfuse observability, online scoring, prompt versioning, anti-extraction defense, and a 17-case hybrid eval suite.

---

## Stack

- **Framework:** Next.js 16 (App Router) on Vercel
- **Styling:** Tailwind 4 + custom CSS for the editorial layout, Lenis for smooth scroll, Framer Motion
- **Chat agent:** `@ai-sdk/react` + `@ai-sdk/anthropic` — Claude Haiku 4.5, streaming, tool use, prompt caching
- **Knowledge retrieval:** in-process embedding RAG (OpenAI `text-embedding-3-small` + cosine similarity) over a structured first-person knowledge base — falls back to keyword dict lookup when embeddings unavailable
- **Observability:** Langfuse (per-stage traces, online scoring on `quality / safety / in_voice`, prompt versioning)
- **Rate limiting:** Upstash Redis sliding-window (5/min, 30/hr, 60/day) with in-memory fallback
- **Notifications:** Telegram (push to phone, owner can reply directly) + Resend (email backup) — both fail-silent if env vars are missing

## The chat agent

The "Ask Umang" widget is a production-grade chat agent grounded in a structured first-person knowledge base. Notable features:

- **4 tools:** `lookup_detail` (embedding RAG), `record_lead` (lead capture → Telegram + email), `record_unknown_question` (logs out-of-scope questions for follow-up), `get_resume_url` (deterministic resume URL)
- **Bounded agentic loop:** `stepCountIs(5)` — model can chain up to 5 tool calls per turn
- **Speculative cache primer:** `/api/chat/warm` fired on panel open so the first real reply gets prompt-cache pricing + faster TTFT
- **Anti-extraction defense:** per-request canary token + post-stream leak detection + system-prompt rules. Suspected leaks tag the trace `prompt-leak-suspected`
- **Conversation cap:** 40 messages → automatic email-handoff response
- **Buttery streaming:** decoupled drain timer (25 ms × 2 chars) — display rate independent of network rate
- **Persistence:** `sessionStorage` + `#chat` URL deep-link (the panel auto-opens on `umanggarg.dev/#chat`)

Private operations dashboard at `/ops` (passcode-gated) showing per-trace cost, latency p50/p99, leak count, jailbreak attempts, leads captured, and online quality scores.

## Local development

```bash
git clone https://github.com/umanggarg28/portfolio
cd portfolio
npm install
cp .env.example .env.local   # see .env.example for what's needed
npm run embed                # one-time: generate knowledge embeddings
npm run dev                  # http://localhost:3000
```

## Available scripts

| Command | Purpose |
|---|---|
| `npm run dev` | Start the Next.js dev server |
| `npm run build` | Production build |
| `npm run lint` | ESLint |
| `npm run embed` | Re-generate `src/content/knowledge-embeddings.json` after editing `knowledge.ts` |
| `npm run eval` | Run the 17-case chat-agent eval suite (deterministic assertions + Sonnet judge). `--case=<id>` to filter, `--no-judge` to skip the LLM judge, `--verbose` for full responses |
| `npm run test:notify` | Smoke-test Telegram + Resend notify channels directly |

## Deploying

Pushes to `main` deploy automatically via Vercel. All env vars are documented in `.env.example`. The chat works in degraded modes when optional services aren't configured (no Langfuse → no traces; no OpenAI key → keyword dict lookup; no Telegram → email-only notifications; etc.).
